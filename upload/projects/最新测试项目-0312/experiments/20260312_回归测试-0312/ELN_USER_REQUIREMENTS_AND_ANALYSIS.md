### 2.4 审核流程设计

```
┌────────┐  提交   ┌────────────┐  通过   ┌────────┐
│ DRAFT  │ ────► │ PENDING    │ ────► │ LOCKED │
│ (草稿) │       │ REVIEW     │       │ (锁定) │
└────────┘       │ (待审核)   │       └────────┘
     ▲           └────────────┘            │
     │                 │ 要求修改          │ 解锁
     │                 ▼                   ▼
     │           ┌────────────┐    ┌──────────────┐
     │           │ NEEDS      │    │ UNLOCKED     │
     │           │ REVISION   │    │ (已解锁)     │
     │           │ (需修改)   │    └──────────────┘
     │           └────────────┘            │
     │                                     │ 重新提交
     └─────────────────────────────────────┘

审核操作类型：
├── SUBMIT - 提交审核（可选择审核人、附加留言）
├── APPROVE - 审核通过（锁定实验）
├── REQUEST_REVISION - 要求修改（附带意见和批注附件）
└── TRANSFER - 转交审核（转给其他项目负责人）
```

### 4.2 开发优先级建议

#### 第一优先级 (P1) - 核心体验完善
1. **文件存储结构重构** - 按项目+实验分层存储
2. **解锁功能** - 申请解锁/直接解锁/解锁历史
3. **项目状态管理** - 已完成/已归档状态转换

#### 第二优先级 (P2) - 功能增强
4. **项目文档管理** - 立项报告/阶段性总结/结题报告
5. **管理员文件目录** - 树形浏览/ZIP下载
6. **锁定PDF生成** - 自动生成标准化PDF

#### 第三优先级 (P3) - 体验优化
7. **批量审核** - 提高审核效率
8. **审核提醒** - 消息通知机制
9. **版本历史** - 实验版本快照

## 第五部分：结论

### 5.1 系统现状总结

BioLab ELN v3.1 已完成核心功能开发，具备以下能力：
- ✅ 完整的用户认证和权限管理体系
- ✅ 项目管理和成员角色分配
- ✅ 实验记录创建、编辑和富文本支持
- ✅ AI智能信息提取
- ✅ 完整度自动评分
- ✅ 规范的审核流程（指定审核人、转交审核）
- ✅ 用户管理功能（管理员专属）
- ✅ 统一的侧边栏和任务管理界面

### 5.2 主要差距

1. **文件管理** - 存储结构需重构，缺乏管理员文件浏览功能
2. **项目生命周期** - 缺少已完成/已归档状态管理
3. **解锁机制** - 已锁定实验无法修改
4. **PDF生成** - 审核通过后无法生成标准化PDF

### 5.3 下一阶段工作

建议按以下顺序推进 v3.2 开发：

```
Week 1: 文件存储结构重构 + 解锁功能
Week 2: 项目状态管理 + 项目文档管理
Week 3: 管理员文件目录 + 锁定PDF生成
Week 4: 测试与优化
```
