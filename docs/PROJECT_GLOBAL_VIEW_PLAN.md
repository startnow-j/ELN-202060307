## 一、需求概述

### 1.1 全局视角功能

**背景**：当前管理员和超级管理员默认具有所有项目的权限，但在项目列表中无法区分"我创建的项目"、"我参与的项目"和"因全局角色可访问的项目"。

**需求**：在项目模块级别提供"全局视角"切换功能：
- **普通视角（默认）**：只显示我创建的项目、我参与的项目
- **全局视角**：显示系统中所有项目（仅管理员/超级管理员可用）

### 1.2 项目基本信息显示修复

**问题**：项目详情页的"项目信息"Tab中，项目描述未显示或显示不正确。

### 1.3 项目基本信息编辑功能

**需求**：在项目详情页的"项目信息"Tab中，允许项目负责人/管理员编辑：
- 开始日期（startDate）
- 预计结束日期（expectedEndDate）
- 项目描述（description）

### 2.2 项目基本信息显示修复

#### 当前问题分析

查看 `ProjectDetail.tsx` 代码，项目描述的显示逻辑存在：

```tsx
{project.description && (
  <div className="pt-4 border-t">
    <p className="text-sm text-muted-foreground mb-2">项目描述</p>
    <p className="text-sm">{project.description}</p>
  </div>
)}
```

**可能的问题**：
1. `project.description` 为空或未正确传递
2. 布局样式问题导致不可见

**修复方案**：
- 检查API返回的数据是否包含description
- 调整样式，确保描述始终显示（即使为空也显示占位符）

#### 2.3.2 编辑状态管理

```typescript
// 编辑模式
const [isEditing, setIsEditing] = useState(false)
const [editForm, setEditForm] = useState({
  startDate: '',
  expectedEndDate: '',
  description: ''
})

// 进入编辑模式
const handleStartEdit = () => {
  setEditForm({
    startDate: project.startDate || '',
    expectedEndDate: project.expectedEndDate || project.endDate || '',
    description: project.description || ''
  })
  setIsEditing(true)
}

// 保存修改
const handleSaveEdit = async () => {
  const res = await fetch(`/api/projects/${project.id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(editForm)
  })
  if (res.ok) {
    setIsEditing(false)
    await refreshData()
  }
}
```
