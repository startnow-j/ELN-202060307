## 一、功能概述

### 1.1 目标

在AI调用过程中，自动对发送给AI的内容进行敏感信息脱敏处理，保护隐私和机密信息不外泄。

### 1.2 核心特性

- **部分脱敏**：保留部分信息，不完全丢失上下文（如"张三"→"张X"）
- **内置规则**：提供常用的脱敏规则，可一键开关
- **自定义敏感词**：管理员可添加实验室特有的敏感词
- **姓名识别配置**：可配置姓名标签和称谓，灵活识别不同场景的人名

### 1.3 数据流向

```
原始文件 ──→ 读取内容 ──→ 【读取云端安全配置】──→ 应用脱敏规则 ──→ 发送给AI
                              ↑
                          云端保存
                        （管理员配置）
```

## 二、内置脱敏规则

### 2.1 规则列表

| 规则名称 | 匹配模式 | 脱敏方式 | 示例 |
|---------|---------|---------|------|
| 身份证号 | `\d{17}[\dXx]` 或 `身份证[\s:：]*\d{17}[\dXx]` | 保留前6后4 | 320102********1234 |
| 手机号 | `1[3-9]\d{9}` 或 `手机[\s:：]*1[3-9]\d{9}` | 保留前3后4 | 138****5678 |
| 邮箱 | `[\w.-]+@[\w.-]+\.\w{2,}` | 保留首字母和域名 | t***@hospital.com |
| 银行卡号 | `\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}` | 保留前4后4 | 6222****5678 |
| 中文姓名 | 标签+姓名 / 姓氏+称谓 | 保留姓氏 | 张三→张X |
| 固定电话 | `\d{3,4}[\s-]?\d{7,8}` | 保留区号 | 010-****5678 |

### 2.2 姓名识别规则

**两种模式**：

1. **标签+姓名模式**
   - 标签：实验人、操作人、负责人、供体、患者、受试者、记录人、审核人、作者、确认人
   - 格式：`标签[\s:：]*[\u4e00-\u9fa5]{2,4}`
   - 示例："负责人：张三" → "负责人：张X"

2. **姓氏+称谓模式**
   - 常用姓氏：王李张刘陈杨黄赵吴周...（约100个）
   - 称谓：先生、女士、小姐、老师、教授、医生、护士、主任、博士、同学
   - 格式：`姓氏+称谓`
   - 示例："王老师" → "王**"

## 三、自定义敏感词配置

### 3.1 配置项

| 字段 | 类型 | 说明 |
|------|------|------|
| 敏感词 | string | 要匹配的关键词 |
| 替换文本 | string | 替换后的文本 |
| 匹配方式 | enum | exact(精确) / fuzzy(模糊，包含即匹配) |
| 是否启用 | boolean | 开关 |

### 3.2 示例

| 敏感词 | 替换文本 | 匹配方式 | 效果 |
|--------|---------|---------|------|
| XX医院 | [医院名称] | fuzzy | "就诊于XX医院" → "就诊于[医院名称]" |
| XX生物 | [供应商] | fuzzy | "购自XX生物公司" → "购自[供应商]公司" |
| 机密 | [机密] | exact | "机密文件" → "[机密]文件" |

## 四、数据结构设计

### 4.1 数据库模型

```prisma
model AISecurityConfig {
  id              String   @id @default(cuid())
  
  // 内置规则配置（JSON格式）
  builtinRules    String   // {"idCard":{"enabled":true,"mode":"partial"},...}
  
  // 自定义敏感词（JSON数组）
  customKeywords  String   // [{"keyword":"XX医院","replacement":"[医院]","matchType":"fuzzy"}...]
  
  // 姓名识别配置（JSON格式）
  nameRecognition String   // {"labels":["实验人","操作人",...],"titles":["先生","女士",...]}
  
  // 元数据
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  @@map("ai_security_config")
}
```

### 4.2 TypeScript 类型定义

```typescript
// 内置规则配置
interface BuiltinRuleConfig {
  enabled: boolean
  mode: 'partial' | 'full'  // partial: 部分脱敏, full: 完全脱敏
}

interface BuiltinRules {
  idCard: BuiltinRuleConfig
  phone: BuiltinRuleConfig
  email: BuiltinRuleConfig
  bankCard: BuiltinRuleConfig
  telephone: BuiltinRuleConfig
  chineseName: BuiltinRuleConfig
}

// 自定义敏感词
interface CustomKeyword {
  id: string
  keyword: string
  replacement: string
  matchType: 'exact' | 'fuzzy'
  enabled: boolean
}

// 姓名识别配置
interface NameRecognitionConfig {
  labels: string[]    // 姓名标签列表
  titles: string[]    // 称谓列表
  surnames: string[]  // 常用姓氏列表（可选自定义）
}

// 完整配置
interface AISecuritySettings {
  builtinRules: BuiltinRules
  customKeywords: CustomKeyword[]
  nameRecognition: NameRecognitionConfig
}
```

## 五、API 设计

### 5.1 获取配置

```
GET /api/ai-security
Response: {
  builtinRules: BuiltinRules,
  customKeywords: CustomKeyword[],
  nameRecognition: NameRecognitionConfig
}
```

### 5.2 更新配置

```
PUT /api/ai-security
Body: {
  builtinRules?: BuiltinRules,
  customKeywords?: CustomKeyword[],
  nameRecognition?: NameRecognitionConfig
}
```

### 5.3 添加敏感词

```
POST /api/ai-security/keywords
Body: {
  keyword: string,
  replacement: string,
  matchType: 'exact' | 'fuzzy'
}
```

### 5.4 删除敏感词

```
DELETE /api/ai-security/keywords/:id
```
