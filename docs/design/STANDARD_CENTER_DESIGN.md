## 一、方案概述

### 1.1 模块定位

将原"实验模板"模块转型为"实验规范中心"，提供实验室文档的集中管理。

| 维度 | 说明 |
|------|------|
| **核心功能** | 文档的上传、分类、查看、下载 |
| **设计原则** | 轻量级、权限分明、无复杂流程 |
| **线上管理** | 文档存放、分类检索、便捷获取 |
| **线下完成** | 文档审核、内容修改、版本更新 |

### 1.2 两大功能区

| 功能区 | 定位 | 权限 |
|--------|------|------|
| **📋 SOP库** | 标准操作规程、参考文献、技术文档 | 管理员管理，所有人查看下载 |
| **📄 模板共享** | Word/Excel模板文件 | 大家共享，上传者和管理员可管理 |

## 五、数据模型

### 5.1 Prisma Schema

```prisma
// 规范文档（SOP和模板统一模型）
model StandardDocument {
  id          String   @id @default(cuid())
  
  // 基本信息
  name        String                    // 文档名称
  docType     DocType                   // 文档类型：SOP 或 TEMPLATE
  
  // 分类和标签
  category    String                    // 分类
  tags        String?                   // 标签，逗号分隔：复苏,培养,操作规范
  description String?                   // 文档简介
  
  // 文件信息
  fileName    String                    // 原始文件名
  filePath    String                    // 存储路径
  fileType    String                    // 文件类型：WORD / EXCEL / PDF
  fileSize    Int                       // 文件大小（字节）
  
  // 上传者
  uploaderId  String
  uploader    User     @relation(fields: [uploaderId], references: [id])
  
  // 统计
  downloadCount Int    @default(0)      // 下载次数
  
  // 时间
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  
  @@map("standard_documents")
}

// 文档类型枚举
enum DocType {
  SOP       // SOP库 - 标准操作规程
  TEMPLATE  // 模板共享 - 记录模板
}
```

### 6.2 权限验证逻辑

```typescript
// 权限检查函数
async function checkDocPermission(
  userId: string, 
  doc: StandardDocument, 
  action: 'create' | 'update' | 'delete'
): Promise<boolean> {
  const user = await db.user.findUnique({ where: { id: userId } })
  
  if (!user || !user.isActive) return false
  
  const isAdmin = user.role === 'ADMIN' || user.role === 'SUPER_ADMIN'
  
  // SOP权限：仅管理员
  if (doc.docType === 'SOP') {
    return isAdmin
  }
  
  // 模板权限：上传者或管理员
  if (doc.docType === 'TEMPLATE') {
    switch (action) {
      case 'create':
        return true // 所有人可创建
      case 'update':
      case 'delete':
        return isAdmin || doc.uploaderId === userId
    }
  }
  
  return false
}
```
