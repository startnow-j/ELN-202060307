> 版本：v1.0 草案
> 状态：待确认
> 创建日期：2025-03-06

---

## 一、方案概述

### 1.1 设计原则

采用**混合方案**：仪表盘（个人工作台）+ 数据中心（管理驾驶舱）两个独立页面

| 页面 | 定位 | 目标用户 | 核心功能 |
|------|------|---------|---------|
| 仪表盘 | 个人工作台 | 所有用户 | 我的待办、智能提醒、个人数据、最近实验 |
| 数据中心 | 管理驾驶舱 | 管理员、超级管理员 | 全局统计、趋势图表、用户活跃度、预警面板 |

### 1.2 架构决策理由

| 考量因素 | 说明 |
|---------|------|
| **职责分离** | 仪表盘关注个人事务，数据中心关注全局监控，互不干扰 |
| **符合现有模式** | 用户管理、文件管理已是管理员独立入口，保持一致性 |
| **管理员双重身份** | 管理员进入系统看个人仪表盘，需要全局视角时进入数据中心 |
| **实现简单** | 无需复杂的视角切换逻辑，权限控制简单 |

### 2.3 模块详细说明

#### 模块1：我的待办

| 字段 | 说明 |
|------|------|
| 数据来源 | 复用"我的任务"模块的数据逻辑 |
| 展示方式 | 四个分类卡片，点击跳转到对应任务列表 |
| 待确认事项 | ❓ 是否显示每个分类的数量徽章？ |

#### 模块2：智能提醒

| 提醒类型 | 触发条件 | 优先级 |
|---------|---------|--------|
| 实验被退回 | 审核状态变为NEEDS_REVISION | 高 |
| 项目即将到期 | 预计结束日期距今≤7天 | 中 |
| 草稿长期未提交 | 草稿状态超过N天（可配置） | 低 |
| 解锁申请已处理 | 申请状态变更 | 高 |

**待确认事项**：
- ❓ 提醒规则是否需要可配置（用户自行设置）？
- ❓ 是否需要"标记已读"功能？
- ❓ 提醒数量上限（显示最近N条）？

#### 模块3：个人数据概览

| 指标 | 计算方式 |
|------|---------|
| 我的实验 | authorId = currentUser 的实验总数 |
| 本月新增 | 本月创建的实验数量 |
| 审核通过率 | 已锁定实验 / 已提交审核的实验 |
| 平均评分 | 我的所有实验的平均完整度评分 |

**待确认事项**：
- ❓ 是否需要环比数据（与上月对比）？
- ❓ 是否需要增加其他指标？

#### 模块4：最近实验

| 配置项 | 说明 |
|--------|------|
| 数据范围 | 我的实验 |
| 排序规则 | 按updatedAt降序 |
| 显示数量 | 最近5条 |
| 展示信息 | 标题、状态、评分、更新时间 |

**待确认事项**：
- ❓ 显示数量是否可配置？

#### 模块5：我的项目

| 配置项 | 说明 |
|--------|------|
| 数据范围 | 我创建或参与的项目 |
| 排序规则 | 按updatedAt降序 |
| 显示数量 | 最近4个 |
| 展示信息 | 项目名、状态、描述、实验数量 |

### 3.4 模块详细说明

#### 模块1：核心指标

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 总实验 | 全部实验数量 | 当前系统实验总数 |
| 本月新增 | 本月创建的实验数 | 反映近期活跃度 |
| 待审核 | PENDING_REVIEW状态数量 | 需要处理的工作量 |
| 通过率 | LOCKED / 已审核 | 审核质量指标 |
| 活跃用户 | 最近7天有登录的用户数 | 用户活跃度 |

**待确认事项**：
- ❓ 指标选择是否需要调整？
- ❓ 是否需要同比/环比数据？
- ❓ 是否需要导出功能？

#### 模块2：实验趋势

| 配置项 | 说明 |
|--------|------|
| 时间维度 | 本周/本月/本季度（可选） |
| 数据系列 | 创建数、提交数、审核通过数 |
| 图表类型 | 柱状图或折线图 |

**待确认事项**：
- ❓ 默认显示哪个时间维度？
- ❓ 是否需要对比功能（与上周/上月对比）？
- ❓ 图表库选型（Chart.js / Recharts / ECharts）？

#### 模块3：项目进度

| 展示内容 | 说明 |
|---------|------|
| 项目名称 | 按实验数量排序 |
| 进度条 | 实验完成率（LOCKED/总数） |
| 状态标识 | 进行中/已完成/延期风险 |

**待确认事项**：
- ❓ 进度计算方式：实验完成率 vs 预计时间进度？
- ❓ 显示项目数量：全部 or Top 5？
- ❓ 是否需要项目时间线视图？

#### 模块4：用户贡献

| 展示内容 | 说明 |
|---------|------|
| 用户排名 | 按实验数量排序 |
| 贡献指标 | 实验数量/通过率/平均评分 |
| 时间范围 | 可选：本周/本月/本季度 |

**待确认事项**：
- ❓ 排名是否公开（所有用户可见）？
- ❓ 排名维度选择？
- ❓ 是否需要用户活跃度趋势图？

#### 模块5：预警面板

| 预警类型 | 触发条件 | 级别 |
|---------|---------|------|
| 待审核积压 | 待审核实验 > N条 或 存在超过N天的 | 🔴 高 |
| 长期未更新 | 草稿/待修改状态超过N天 | 🟡 中 |
| 项目延期风险 | 预计结束日期已过但仍未完成 | 🟡 中 |
| 解锁申请积压 | 待处理解锁申请 > N条 | 🟡 中 |

**待确认事项**：
- ❓ 预警阈值是否可配置？
- ❓ 预警是否需要推送通知（邮件/站内）？
- ❓ 历史预警记录是否需要保存？

### 4.3 数据库扩展（可选）

如果需要支持提醒已读状态、预警记录等功能：

```prisma
// 用户提醒记录
model UserReminder {
  id          String   @id @default(cuid())
  userId      String
  user        User     @relation(...)
  
  type        String   // REMINDER_TYPE 枚举值
  title       String
  content     String
  relatedId   String?  // 关联的实验/项目ID
  isRead      Boolean  @default(false)
  
  createdAt   DateTime @default(now())
  
  @@map("user_reminders")
}

// 系统预警记录（可选）
model SystemAlert {
  id          String   @id @default(cuid())
  type        String
  level       String   // HIGH, MEDIUM, LOW
  content     String
  relatedId   String?
  isResolved  Boolean  @default(false)
  
  createdAt   DateTime @default(now())
  resolvedAt  DateTime?
  
  @@map("system_alerts")
}
```

### 4.4 图表库选型

| 候选库 | 优点 | 缺点 |
|--------|------|------|
| **Recharts** | React友好、轻量、TypeScript支持好 | 图表类型较少 |
| **Chart.js** | 成熟稳定、社区大 | 需要react-chartjs-2包装 |
| **ECharts** | 功能强大、图表丰富 | 体积较大 |

**推荐**：Recharts（轻量、React生态、适合当前需求）
