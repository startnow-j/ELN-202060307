2）完善目前项目中已完成和已归档的业务逻辑
已完成 (COMPLETED)
含义：项目正常结束，所有实验工作已完成
触发条件：项目负责人手动标记
业务影响：
关联的实验记录自动锁定（如有未完成的实验需提醒）
可生成项目总结报告
成员仍可查看、导出数据
可随时重新激活为"进行中"
已归档 (ARCHIVED)
含义：项目长期封存，不再活跃操作
触发条件：只能从"已完成"状态转换，或管理员强制归档
业务影响：
项目在默认列表中隐藏（需筛选查看）
成员权限降为只读
数据仅保留，不做修改
可能涉及数据迁移/压缩存储
是否需要完善这个设计？

状态转换规则（ACTIVE → COMPLETED → ARCHIVED）
| 转换 | 条件 | 权限 |
|------|------|------|
| ACTIVE → COMPLETED | 关联实验全部锁定或有提醒 | 项目负责人/管理员 |
| COMPLETED → ACTIVE | 无限制 | 项目负责人/管理员 |
| COMPLETED → ARCHIVED | 无限制 | 项目负责人/管理员 |

| ACTIVE → ARCHIVED | 强制归档 | 仅管理员 |

记录4：实验记录和审核的一些优化
1）实验记录保存后不退出界面
保存按钮：保存成功后保持当前编辑界面，可继续编辑
取消按钮：点击左上角返回按钮才退出编辑器
提交审核：成功后才退出编辑器
2）提交审核功能时，要求能选择一个审核人，并且进行留言，然后提交审核
a.需要建立审核人选择逻辑，仅显示关联项目的负责人（项目创建者 + PROJECT_LEAD 成员）+ 系统管理员。且要求自己不能选择自己为审核人。

b.创建提交审核对话框组件，支持审核人选择和留言功能
留言功能：可选填写提交留言给审核人查看
分组显示：按项目分组展示审核人
留言也在审核记录中显示

c.提交审核后，审核人在项目审核快捷界面上能够看到审核需求和留言。
d.审核操作面板 (ReviewActionPanel)上具有平行选项
审核意见：填写审核反馈
附件上传：支持上传批注文件（PDF、Word、Excel）
审核转交：可选择其他该项目的有权审核人进行转交
要求修改：弹出增强面板，要有"选择批注文件"下拉区，显示该实验已有附件列表，可勾选并下载；还要支持拖拽上传新批注文件；填写修改意见。提交后实验人收到通知，在实验详情页面上显示修改意见和附件。
审核通过：可选填写审核意见，通过后锁定实验记录，不能再进行修改。
