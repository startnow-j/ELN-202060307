## 二、视角模式定义

### 概念定义

| 视角模式 | 适用角色 | 数据范围 | 使用场景 |
|---------|---------|---------|---------|
| **默认视角** | 所有用户 | 仅显示用户参与的实验/项目 | 日常工作，专注自己的工作内容 |
| **全局视角** | 管理员、超级管理员 | 显示系统中所有实验/项目 | 监督管理、审核审批、全局查看 |

### 适用角色矩阵

```
┌─────────────────────────────────────────────────────────────┐
│                        角色权限矩阵                           │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│   角色      │  默认视角   │  全局视角   │      备注          │
├─────────────┼─────────────┼─────────────┼───────────────────┤
│ 研究员      │     ✅      │     ❌      │ 只能使用默认视角    │
│ 管理员      │     ✅      │     ✅      │ 可切换使用          │
│ 超级管理员  │     ✅      │     ✅      │ 可切换使用          │
└─────────────┴─────────────┴─────────────┴───────────────────┘
```

## 三、各模块视角差异详解

### 3.1 实验记录模块

| 功能 | 默认视角 | 全局视角 |
|------|---------|---------|
| 实验列表 | 显示自己创建的 + 所属项目的实验 | 显示所有实验 |
| 编辑权限 | 仅可编辑自己的实验 | 可编辑所有实验 |
| 删除权限 | 仅可删除自己的草稿 | 可删除任何实验 |
| 搜索范围 | 仅搜索可见范围内的实验 | 搜索全系统实验 |

**数据过滤逻辑**：

```typescript
// 默认视角
where: {
  OR: [
    { authorId: userId },                           // 自己创建的
    { experimentProjects: {                         // 所属项目的
      some: { project: { projectMembers: { some: { userId } } } }
    }}
  ]
}

// 全局视角（管理员）
where: {}  // 无过滤，返回所有
```

### 3.2 项目管理模块

| 功能 | 默认视角 | 全局视角 |
|------|---------|---------|
| 项目列表 | 显示自己创建的 + 参与的项目 | 显示所有项目 |
| 项目详情 | 可查看参与的项目 | 可查看所有项目 |
| 成员管理 | 仅可管理自己负责的项目 | 可管理所有项目 |
| 项目状态 | 可操作参与的项目 | 可操作所有项目 |

**特殊说明**：
- 超级管理员可删除任何项目
- 超级管理员可恢复已归档项目

### 3.3 我的任务模块

| 任务类型 | 默认视角 | 全局视角 |
|---------|---------|---------|
| 我的草稿 | 自己的草稿 | 不变（始终是自己的） |
| 待我审核 | 作为项目负责人的待审核 + 被指定为审核人 | 增加：全局待审核实验（可主动审核） |
| 需要修改 | 自己被退回的实验 | 不变（始终是自己的） |
| 解锁申请 | 自己的申请 | 增加：待处理的他人解锁申请 |

**关键差异**：
- 全局视角下，管理员可以看到更多待审核任务
- 管理员可以主动审核任何待审核的实验

### 3.4 审核流程模块

| 功能 | 默认视角 | 全局视角 |
|------|---------|---------|
| 提交审核 | 只能提交自己的实验 | 可以代理提交任何实验 |
| 审核操作 | 只能审核指定/项目的实验 | 可以审核任何待审核实验 |
| 解锁审批 | 项目负责人的解锁申请 | 可审批任何解锁申请 |
| 审核历史 | 查看自己参与的审核 | 查看所有审核历史 |

## 四、技术实现说明

### 4.1 API 参数设计

```typescript
// 请求参数
GET /api/experiments?viewMode=default|global

// 后端处理
if (viewMode === 'global' && await isAdmin(userId)) {
  // 返回所有数据
  return await db.experiment.findMany()
} else {
  // 返回用户可见范围的数据
  return await db.experiment.findMany({
    where: buildUserScopeFilter(userId)
  })
}
```

### 4.2 前端状态管理

```typescript
// AppContext.tsx
interface AppState {
  viewMode: 'default' | 'global'
  setViewMode: (mode: 'default' | 'global') => void
}

// 使用
const { viewMode, setViewMode } = useApp()

// API 请求时携带
const { data } = useQuery({
  queryKey: ['experiments', viewMode],
  queryFn: () => authFetch(`/api/experiments?viewMode=${viewMode}`)
})
```

### 4.3 视角切换组件

位置：Header 组件右上角

```tsx
{isAdmin && (
  <div className="flex items-center gap-2">
    <Switch
      checked={viewMode === 'global'}
      onCheckedChange={(checked) => setViewMode(checked ? 'global' : 'default')}
    />
    <Label>全局视角</Label>
  </div>
)}
```
